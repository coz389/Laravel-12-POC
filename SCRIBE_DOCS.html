1. Overview

This document provides the technical specifications for the Laravel 12 API. Documentation is generated using Scribe, which extracts information from routes and controller annotations to produce human-readable HTML/Blade documentation.

Base URL

http://localhost:8000/api

Documentation URL

http://localhost:8000/docs

2. Installation & Scribe Setup

To initialize documentation in a fresh environment:

Bashcomposer require knuckleswtf/scribe --dev
php artisan vendor:publish --provider="Knuckles\Scribe\ScribeServiceProvider" --tag=scribe-config

This creates the config/scribe.php file used to manage documentation behavior.

4. Authentication APIs

Authentication is handled via Laravel Sanctum. All protected routes require a Bearer Token.

4.1 User Login

POST /login

Authenticate credentials and receive a bearer token.

Request Body

ParameterTypeRequiredExample
emailstringYesjohn@example.com
passwordstringYespassword123

Response (200 OK)

JSON{
  "status": true,
  "message": "Login successful",

}

4.2 User Logout

POST /logout

Revokes the current user's session token.

Requirement: @authenticated header.

6. Generating Documentation

Scribe documentation is not updated automatically. Run the following command after making changes to your controllers or routes:

Terminalphp artisan scribe:generate

This command performs the following:

  • Analyzes routes/api.php.
  • Extracts DocBlock comments and Attributes.
  • Generates resources/views/scribe/index.blade.php.
  • Outputs Postman and OpenAPI specs in storage/app/private/scribe/.

7. Best Practices & Annotations

  • @group: Always categorize endpoints (e.g., @group Authentication).
  • @bodyParam: Use this to define expected input. Scribe will automatically generate "Try It Out" fields.
  • @response: Provide manual response examples for complex logic.
  • @authenticated: Mark routes that require a Sanctum token.
